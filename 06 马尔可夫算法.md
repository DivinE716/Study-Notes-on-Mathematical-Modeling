# 马尔可夫算法

理论核心：系统的当前状态只和上一个状态有关，和其它历史状态无关，又称“无记忆性”。

## 1. 马尔可夫链

一个马尔可夫链包含以下要素：
1. 状态空间 $S = \{s_1, s_2, ..., s_n\}$
   在天气例子中， $S =$ { 晴天, 阴天, 雨天 }
2. 转移概率 $p_{ij} = P(X_{t+1} = j | X_t = i)$
3. 转移概率矩阵 $P = [p_{ij}]$（一个 $n\times n$的矩阵， $p_{ij}$表示第 $i$个状态转移到第 $j$个状态的概率，因此该矩阵每一行/列的和都为1）
4. 初始分布 $\pi(0) = [\pi_1(0), \pi_2(0), ..., \pi_n(0)]$

## 2. 马尔可夫预测算法

### 一、基本思想

#### 1）状态定义 
将研究对象所有可能出现的、互斥且完备的情况划分成有限个离散的状态。

#### 2）转移矩阵估计 
量化系统从一个状态转移到另一个状态（或保持不变）的可能性。

#### 3）初始状态分布 
指定预测开始时（时间t=0），系统处于各个状态的概率

#### 4）根据状态转移概率矩阵进行预测
利用初始状态分布和转移概率矩阵，计算在未来任意时间点 n（n=1,2,3,…），系统处于各个状态的概率分布。

---

### 二、建模流程

#### step 1：确定问题
    1. 系统是否有明确的、可区分的状态？
    2. 未来状态是否主要依赖于当前状态？
    3. 转移概率是否相对稳定？

#### step 1：状态定义
    1. 状态要互斥且完备（每个观察必须属于且只属于一个状态）
    2. 状态数量要适中（太少失去精度，太多增加复杂性）
    3. 状态要有实际意义

#### step 1：确定转移概率矩阵
    1. 收集历史数据，统计系统在状态间转移的次数。
    2. 计算从状态\( S_i \)转移到状态\( S_j \)的概率\( P_{ij} \)。

#### 第四步：初始化状态分布

#### 第五步：模型预测和验证
    1. 检查矩阵的数学性质
    2. 与历史数据对比
    3. 交叉验证
    4. 检验马尔可夫性假设
